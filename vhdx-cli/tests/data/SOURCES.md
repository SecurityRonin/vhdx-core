# VHDX Test Data — Provenance

Real VHDX and VHD images from independent third-party sources.

These exist to catch parser bugs that synthetic fixtures (built by this library)
cannot detect — a doer-checker violation where both the fixture builder and the
parser encode the same incorrect assumption.

All files verified against the checksums below.

### qemu_fixed.vhdx
- **Origin**: Generated locally with QEMU v11.0.0 (Homebrew)
- **Command**: `qemu-img create -f vhdx -o subformat=fixed qemu_fixed.vhdx 8M`
- **Virtual size**: 8 MiB (8,388,608 bytes)
- **File size**: 16 MiB (16,777,216 bytes) — fixed provisioning: all blocks preallocated
- **SHA-256**: `918b40b57e9b2302e6c178fa6141cd6da369231d8e3a42a955f4840d27acee71`
- **Notes**: Fixed-provisioning VHDX. All BAT payload entries are in FULLY PRESENT
  state and all SectorBitmap entries are allocated — a structurally different BAT
  layout from dynamic images where absent blocks have NO PAYLOAD state.

## Re-downloading

```bash
cd tests/data

# dfvfs images
BASE=https://github.com/log2timeline/dfvfs/raw/main/test_data
curl -OL $BASE/ext2.vhdx
curl -OL $BASE/fat-parent.vhdx
curl -OL $BASE/fat-differential.vhdx
curl -OL $BASE/ext2.vhd

# QEMU-generated images
qemu-img create -f vhdx qemu_empty_dynamic.vhdx 16M
qemu-img create -f vhdx -o subformat=fixed qemu_fixed.vhdx 8M

# Verify checksums
shasum -a 256 *.vhdx *.vhd
```
